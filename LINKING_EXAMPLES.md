## Link Flow Summary

### Authority Flow Direction
```
OUTER CLUSTERS ──────────► CORE HUBS ──────────► CONVERSIONS
(Many pages)              (6 hubs)              (Equipment,
(Authority building)      (SEO power)            Programs)
```

### Page Link Counts

**Homepage:**
- Links out to: All 6 core hubs + 7 cluster hubs = 13 links

**Core Hub Pages:**
- Links out to: 5-7 spokes + 5 other hubs = ~10-12 links
- Links in from: Homepage + related spokes + outer clusters = ~15-30 links

**Core Spoke Pages:**
- Links out to: Parent hub + 2-3 peer spokes = 3-4 links
- Links in from: Parent hub + outer clusters = ~5-10 links

**Outer Cluster Topics:**
- Links out to: 2-4 core hub/spoke pages = 2-4 links
- Links in from: Cluster hub page = 1 link

## Internal Link Density

### Ideal Internal Links Per Page

**Homepage:** 10-15 links
- Navigation: 6 core hubs
- Feature grids: All hubs and clusters

**Core Hubs (Quality Nodes):** 12-15 links
- All spoke pages
- Peer hub pages
- Related clusters

**Core Spokes:** 4-8 links
- Parent hub
- Peer spokes
- Related core pages

**Outer Cluster Topics:** 2-4 links
- Focus on linking TO core
- Minimal peer linking

## Key Principles Applied

### 1. One-Way Flow (Outer → Core)
Outer cluster pages link TO core pages, building authority for core pages.

### 2. Hub-and-Spoke
Each hub serves as a table of contents for its spokes, and spokes link back to their hub.

### 3. Peer Linking
Related hub pages link to each other (progressions ↔ programs ↔ guide)

### 4. Natural User Journeys
Links follow logical user thought processes:
- "I want to learn" → Guide
- "I need a program" → Programs
- "What equipment?" → Equipment
- "Why do this?" → Benefits
- "How to avoid injury?" → Safety

### 5. Strategic CTAs
High-value pages (programs, equipment) are consistently linked from informational pages.

This internal linking structure maximizes:
- SEO value (authority flows to money pages)
- User experience (natural journeys)
- Conversions (guided to monetization)
- Crawlability (every page is 3 clicks from homepage)
